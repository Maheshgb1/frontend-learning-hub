Angular Lifecycle Hooks

Angular lifecycle hooks are methods that let us run code at specific moments in a component’s life — from creation to destruction.

Hook Called When Runs How Often Used For Example Use Case
constructor Component class is created Once Dependency Injection Inject services
ngOnChanges Input() value changes Many times React to input changes Update UI when parent data changes
ngOnInit After first ngOnChanges Once Initialization logic API calls, default data
ngDoCheck Every change detection Many times Custom change detection Deep object comparison
ngAfterContentInit ng-content initialized Once Projected content setup Access projected data
ngAfterContentChecked Projected content checked Many times Validate projected content Log content updates
ngAfterViewInit View & child views ready Once DOM access Focus input, load charts
ngAfterViewChecked View checked Many times Post-view checks Debug UI rendering
ngOnDestroy Before component removal Once Cleanup Unsubscribe, clear timers


Directives

Directives are instructions that tell Angular how to change the DOM (structure, behavior, or appearance).
There are 4 types of directives:

  1. Component Directives: Every component is a directive with a template.
  2. Structural Directives: They add / remove elements from the DOM. Identified by '*' --- *ngIf, *ngFor, *ngSwitch, *ngTemplateOutlet
  3. Attribute Directives: They modify existing elements. Identified by '[attribute]' (ngClass – Add/Remove CSS Classes, ngStyle – Add/Remove CSS Styles, ngModel – Two-Way Binding (Forms))

Pipes

A pipe is used to transform data in the template (HTML) without changing the original value.

  1. Built-in Pipes: Angular provides many ready-to-use pipes.
    Examples: date, uppercase, lowercase, currency, percent, decimal, slice, json, async etc...
  2. Custom Pipes: When built-in pipes are not enough, we create custom pipes.
    Examples: Reverse a string, Mask phone numbers, Convert status codes to text, Custom filtering or formatting
